## English

### Source of truth

`prompt/` and `schema/` at the repo root are the **single source of truth** for Cave Prompt's runtime behaviour. Both Python and TypeScript packages bundle copies of these files — never edit the copies directly.

```
cave_prompt/
├── prompt/cave_prompt.md          ← edit here
├── prompt/cave_prompt.min.md      ← edit here
├── schema/semantic-analysis.schema.json  ← edit here
├── schema/optimized-ir.schema.json       ← edit here
├── packages/python/src/cave_prompt/_data/   ← generated by sync
└── packages/typescript/assets/              ← generated by sync
```

### Making changes to the prompt or schemas

1. Edit the file in `prompt/` or `schema/`.
2. Re-sync both packages:

```bash
# Python
python packages/python/scripts/sync_assets.py

# TypeScript
cd packages/typescript && npm run sync
```

3. Run the test suites — drift-guard tests will fail if bundles are out of sync:

```bash
cd packages/python && pytest
cd packages/typescript && npm test
```

### Running tests

**Python (requires Python 3.10+):**

```bash
cd packages/python
pip install -e ".[dev]"
pytest                     # all tests
pytest tests/test_compiler.py -v   # specific suite
```

**TypeScript (requires Node 18+):**

```bash
cd packages/typescript
npm install
npm run build
npm test
```

### Commit conventions

- `feat(py):` / `feat(ts):` — new feature in a specific package
- `feat:` — cross-package or shared asset change
- `fix(py):` / `fix(ts):` — bug fix
- `docs:` — documentation only
- `test:` — tests only
- `chore:` — tooling, config, scaffolding

### What belongs where

| Change | Where to make it |
|---|---|
| Runtime behaviour, pipeline, config defaults | `prompt/cave_prompt.md` + sync |
| Compressed variant | `prompt/cave_prompt.min.md` + sync |
| IR/output schema | `schema/*.json` + sync |
| CLI flags, SDK API | `packages/python/src/cave_prompt/` or `packages/typescript/src/` |
| Claude Code skill | `skills/cave_prompt/SKILL.md` (standalone, no sync needed) |
| Examples | `examples/` |

### Publishing (maintainers only)

```bash
# Python → PyPI
cd packages/python
python -m build
twine upload dist/*

# TypeScript → npm
cd packages/typescript
npm run build
npm publish
```

## Tiếng Việt

### Nguồn chân lý

`prompt/` và `schema/` ở gốc repo là **nguồn chân lý duy nhất** cho hành vi runtime của Cave Prompt. Cả hai package Python và TypeScript đều bundle bản sao — **không sửa trực tiếp vào bản copy**.

### Thay đổi prompt hoặc schema

1. Sửa file trong `prompt/` hoặc `schema/`.
2. Sync lại cả hai package:

```bash
# Python
python packages/python/scripts/sync_assets.py

# TypeScript
cd packages/typescript && npm run sync
```

3. Chạy test — drift-guard test sẽ fail nếu bundle bị lệch:

```bash
cd packages/python && pytest
cd packages/typescript && npm test
```

### Chạy test

**Python (cần Python 3.10+):**

```bash
cd packages/python
pip install -e ".[dev]"
pytest                     # toàn bộ
pytest tests/test_compiler.py -v   # chỉ compiler
```

**TypeScript (cần Node 18+):**

```bash
cd packages/typescript
npm install
npm run build
npm test
```

### Quy ước commit

- `feat(py):` / `feat(ts):` — tính năng mới trong package cụ thể
- `feat:` — thay đổi cross-package hoặc shared asset
- `fix(py):` / `fix(ts):` — bug fix
- `docs:` — chỉ documentation
- `test:` — chỉ test
- `chore:` — tooling, config, scaffold

### Phân công theo loại thay đổi

| Thay đổi | Làm ở đâu |
|---|---|
| Hành vi runtime, pipeline, config default | `prompt/cave_prompt.md` rồi sync |
| Bản nén | `prompt/cave_prompt.min.md` rồi sync |
| Schema IR/output | `schema/*.json` rồi sync |
| CLI flag, SDK API | `packages/python/` hoặc `packages/typescript/` |
| Claude Code skill | `skills/cave_prompt/SKILL.md` (standalone) |
| Ví dụ | `examples/` |
